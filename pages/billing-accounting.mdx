## Billing Workflow

1. Job completion triggers auto-generation of a draft invoice
2. Invoice line items calculated from client's rate card
3. Administrator reviews and approves
4. PDF generated and stored to MinIO (S3-compatible)
5. Approved invoice synced to Xero via accounting-integration (port 3018)
6. Payment status tracked via Xero polling

## Invoice Fields

| Field | Description |
|---|---|
| `invoice_number` | Auto-generated unique ID |
| `client_id` | Linked client organisation |
| `job_id` | Originating transfer job |
| `line_items` | Array of `{description, amount, vat_rate}` |
| `subtotal` | Sum of line item amounts |
| `vat_amount` | Calculated VAT |
| `total_amount` | Subtotal + VAT |
| `status` | `draft → approved → synced → paid` (or `failed`/`overdue`) |
| `due_date` | Payment due date |
| `pdf_s3_key` | S3/MinIO key for generated PDF |

## Invoice Status Machine

```
draft → approved → synced → paid
                 ↘ failed
                 ↘ overdue
```

## Rate Cards

Rate cards define billing per client:
- `billing_type`: `flat_rate`, `per_mile`, `per_hour`, or `combined`
- `rates`: JSON with rates per priority (routine, urgent, emergency)
- `effective_from` / `effective_to`: date range validity

Each priority level can have independent `flat`, `per_mile`, and `per_hour` rates.

## Xero Integration (accounting-integration, port 3018)

The accounting-integration service manages Xero OAuth 2.0 tokens via `xero-token-manager.ts`
and syncs data via `xero-connector.ts`.

| Endpoint | Purpose |
|---|---|
| `POST /v1/accounting/sync/invoice` | Push approved invoice to Xero |
| `POST /v1/accounting/sync/contact` | Sync client as Xero contact |
| `GET /v1/accounting/payment-status/:ref` | Check payment status |
| `GET /v1/accounting/oauth/callback` | OAuth 2.0 callback |
| `GET /v1/integration/health` | Integration health dashboard |

## Timesheet → HR Provider → Payroll

When a PTF closes, the shift check-in/out data flows to the configured HR provider for payroll:

```
PTF Closed → Timesheet Generated → HR Approval → Factorial/Deel Shift Sync → Payroll
```

## Timesheet Approval Page

The `/timesheets` page (administrator, hr_manager) provides:
- List of timesheet periods with status filters (pending, approved, rejected, synced)
- Aggregated KPIs: scheduled vs worked minutes, overtime, break time
- Bulk approve/reject actions
- One-click sync to Factorial/Deel after approval

## Multi-Leg Journey Billing

For multi-leg journeys, each leg can generate separate billing:
- Accommodation costs tracked via `JobAccommodation`
- Meal expenses via `JobMeal`
- Additional expenses (fuel, tolls, parking) via `JobExpense`
- All expenses rolled into the invoice line items

## Client Portal Billing View

Clients accessing `/bookings` can view:
- Invoice history for their organisation
- Payment status per invoice
- Download approved PDFs
