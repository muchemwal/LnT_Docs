## Infrastructure Requirements

| Component | Requirement |
|---|---|
| PostgreSQL | 16+ (primary on 5432, audit on 5433) |
| Redis | 7+ (caching, pub/sub, streams) |
| MinIO | S3-compatible object storage (signatures, PDFs, documents) |
| Keycloak | 24+ for OIDC identity management (PKCE S256) |
| Kong | 3.7 API Gateway (DB-less declarative config) |
| Node.js | 20+ LTS |
| Docker | Container runtime |
| npm | 10.5+ (package manager) |
| Turborepo | 2.0+ (monorepo orchestration) |

## Deployment Options

<Callout type="info">
  The platform supports two deployment strategies: Kubernetes (AWS EKS) for production scale,
  and single-EC2 (Docker Compose + Caddy) for smaller deployments.
</Callout>

### Option 1 — Kubernetes (AWS EKS)

Full Kubernetes deployment with 26 YAML manifests covering all services and infrastructure:
- Service mesh via Linkerd (mTLS between services)
- Horizontal pod autoscaling
- Persistent volume claims for databases
- Ingress via Kong

### Option 2 — Single EC2 (Docker Compose + Caddy)

Suitable for smaller deployments on a single `t3.medium` instance (eu-west-2):
- All services run via Docker Compose
- Caddy reverse proxy with automatic Let's Encrypt HTTPS
- Simplified operational overhead

## Docker Compose (Local Development)

```bash
# Start infrastructure
docker compose up -d

# Run all services
npm run dev
```

Infrastructure services:
- `postgres-primary` — Primary database (port 5432)
- `postgres-audit` — Audit database (port 5433)
- `redis` — Cache + pub/sub (port 6379)
- `minio` — S3-compatible storage (API: 9000, Console: 9001)
- `keycloak` — Identity provider (port 8080)
- `kong` — API Gateway (HTTP: 8000, HTTPS: 8443, Admin: 8001)
- `compliance-service` — Docker-only service (port 3016)

## Screenshot Capture Tool

The `ptf-doc-capture.sh` script guides through capturing all platform screenshots:

```bash
chmod +x docs/ptf-doc-capture.sh && ./docs/ptf-doc-capture.sh
```

Screenshots are saved to `docs/images/` and referenced by both the LaTeX documentation
and this Nextra site.

<Callout>
  After capturing screenshots, copy them to `apps/docs/public/images/` for the Nextra site:
  `cp docs/images/*.png apps/docs/public/images/`
</Callout>

## Building the Docs

```bash
# Development
cd apps/docs && npm run dev

# Production build
cd apps/docs && npm run build && npm run start
```

The docs site runs on port 3100 by default.

## Health Checks

All services expose `GET /health` endpoints. The local health-check script tests ports
3001–3015, 3017, 3018 (excluding compliance-service which is Docker-only).

```bash
# Check all local services
./scripts/health-check.sh
```
