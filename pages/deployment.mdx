# Deployment Guide

import { Callout } from 'nextra/components'

## Infrastructure

Single OVH Public Cloud instance (c2-15, 15 GB RAM):
- All services run in Docker Compose
- Domain set via `DOMAIN` env var in `.env.prod`

| Component | Container | Port |
|---|---|---|
| Caddy (HTTPS proxy) | landtts-caddy | 80, 443 |
| Kong (API gateway) | landtts-kong | 8000 |
| Web App | landtts-web | 3000 |
| API (all services) | landtts-api | 3001-3018 |
| PostgreSQL Primary | landtts-pg | 5432 |
| PostgreSQL Audit | landtts-pg-audit | 5432 |
| Redis | landtts-redis | 6379 |
| MinIO (S3) | landtts-minio | 9000 |
| Keycloak | landtts-keycloak | 8080 |

## Deployment Options

The platform runs as a single Docker Compose stack. All 18 API services are run via `concurrently` in a single API container.

## WebSocket Routing (Caddyfile)

```
/ws/gps       → api:3004  (GPS streaming from mobile)
/ws/dispatch  → api:3002  (Dispatch board live updates)
/ws/rota      → api:3006  (Rota canvas real-time sync)
/auth/*       → keycloak:8080
/v1/*         → kong:8000
/*            → web:3000
```

## Deploying Changes

### Quick deploy (file copy + restart)

```bash
# Copy file to server
scp -i ~/.ssh/id_ed25519 <local-path> ubuntu@<server-ip>:/opt/landtts/<path>

# Rebuild and restart specific service(s)
ssh ubuntu@<server-ip> "cd /opt/landtts/infra/aws-single && docker compose -f docker-compose.prod.yml build api web && docker compose -f docker-compose.prod.yml up -d api web"
```

### Full rebuild (no-cache)

```bash
ssh ubuntu@<server-ip> "cd /opt/landtts/infra/aws-single && docker compose -f docker-compose.prod.yml build --no-cache api web && docker compose -f docker-compose.prod.yml up -d"
```

### Database operations

```bash
# Run Prisma push
docker exec landtts-api npx prisma db push --schema=packages/db/prisma/schema.prisma

# Direct SQL
docker exec landtts-pg psql -U landtts_app -d landtts
```

## Docker Compose (Local Development)

```bash
# Start infrastructure
docker compose up -d

# Run all services
npm run dev
```

Infrastructure services:
- `postgres-primary` — Primary database (port 5432)
- `postgres-audit` — Audit database (port 5433)
- `redis` — Cache + pub/sub (port 6379)
- `minio` — S3-compatible storage (API: 9000, Console: 9001)
- `keycloak` — Identity provider (port 8080)
- `kong` — API Gateway (HTTP: 8000, HTTPS: 8443, Admin: 8001)
- `compliance-service` — Docker-only service (port 3016)

## Screenshot Capture Tool

The `ptf-doc-capture.sh` script guides through capturing all platform screenshots:

```bash
chmod +x docs/ptf-doc-capture.sh && ./docs/ptf-doc-capture.sh
```

Screenshots are saved to `docs/images/` and referenced by both the LaTeX documentation
and this Nextra site.

## Building the Docs

```bash
# Development
cd apps/docs && npm run dev

# Production build
cd apps/docs && npm run build && npm run start
```

The docs site runs on port 3100 by default.

## Health Checks

All services expose `GET /health` endpoints. The local health-check script tests ports
3001–3015, 3017, 3018 (excluding compliance-service which is Docker-only).

```bash
# Check all local services
./scripts/health-check.sh
```
