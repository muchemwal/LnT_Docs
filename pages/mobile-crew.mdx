# Mobile Crew Workflow

import { Callout, Steps } from 'nextra/components'

The React Native (Expo SDK 50) mobile app is the primary interface for crew members during transfers.
It supports **offline-first operation** via WatermelonDB, **biometric authentication**, and
**intelligent trip assistance**.

## Authentication & Onboarding

- **Keycloak OIDC** with PKCE S256 (`landtts-mobile` client)
- **Biometric auth**: Face ID / Touch ID / Fingerprint via `react-native-keychain`
- **Onboarding flow**: First-launch walkthrough for new crew members
- **Location disclosure**: Google Play compliance screen explaining GPS usage

Crew records journey log entries during transport:

| Log Type | Purpose |
|---|---|
| `pickup_event` | Pickup-related observations |
| `journey_update` | General progress / route notes |
| `restraint_event` | Restraint used (type, duration, justification) |
| `delay_event` | Delay with reason |

## GPS Tracking

- Location updates every **5 seconds** during active transport (foreground) / 15 seconds (background)
- Native WebSocket streaming via `wss://domain/ws/gps`
- Android: foreground service with persistent notification ("Tracking for dispatch")
- Each update: latitude, longitude, accuracy (m), heading (°), speed (km/h), device ID
- **Offline buffering**: queues up to 500 points in memory, exponential backoff reconnection (2s → 60s max), batch replay on reconnect
- History: `GET /v1/gps/positions/:jobId`
- Latest: `GET /v1/gps/latest/:crewId`

![GPS Tracking](/images/39-mobile-gps-tracking.png)

## Trip Share (Live Tracking Links)

Crew or dispatchers can generate **public live tracking links** (similar to Uber/Bolt):
- `POST /v1/jobs/:id/share` generates a 24-hour token
- Public page at `/track/:token` shows real-time position on map
- No authentication required for viewers
- Automatically expires after 24 hours or on job completion

## Offline Sync (WatermelonDB)

The mobile app operates **offline-first** with automatic synchronisation:

| Component | Purpose |
|---|---|
| `SyncEngine` | Manages bidirectional sync between local DB and server |
| Schema | WatermelonDB schema mirroring key server models |
| Models | Local model definitions with sync adapters |
| Reconnection | Automatic push of buffered data when connectivity returns |

<Callout type="info">
  GPS locations, journey logs, and status transitions captured offline are queued locally
  and pushed to the server on reconnection. The server reconciles timestamps to maintain
  accurate audit trails.
</Callout>
