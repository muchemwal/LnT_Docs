## System Description

LANDTTS is a microservices-based platform comprising **19 independently deployable services**, a React
web application for dispatchers and administrators, and a React Native (Expo) mobile application for
crew members. All services communicate through a central API gateway and share a PostgreSQL database
via Prisma ORM.

The platform supports five user roles: `administrator`, `dispatcher`, `crew_member`, `hr_manager`,
and `client`. Each role has precisely scoped permissions enforced at the API middleware layer through
Keycloak-issued JWT tokens.

## User Roles

| Role | Description |
|---|---|
| `administrator` | Full platform access. Manages users, settings, billing, compliance. |
| `dispatcher` | Creates bookings, assigns crew/vehicles, monitors dispatch board. |
| `crew_member` | Mobile-focused. Accepts jobs, records journey events, captures signatures. |
| `hr_manager` | Manages employees, certifications, shifts, timesheets, absences. |
| `client` | External portal. Views booking history, submits requests, duplicates bookings. |

## High-Level Architecture

```
Presentation ─── React Web (Vite) + React Native Mobile (Expo SDK 50)
       │
  API Gateway ─── Port 8000, routes by URL prefix
       │
  Service Layer ── 19 microservices (ports 3001–3018)
       │
  Data Layer ───── PostgreSQL (Prisma) + Redis (cache/pub-sub) + S3 (binary)
       │
  Identity ─────── Keycloak OIDC with TOTP MFA
       │
  Integrations ─── Xero │ Factorial HR │ Mapbox
```

## Operational Workflows

- Routine patient transfers (scheduled, non-emergency)
- Urgent and emergency transfers with priority escalation
- Mental Health Act section transfers with legal documentation
- Multi-leg journeys with accommodation, meals, and expense tracking
- Return leg transfers within the same booking
- Client-initiated booking requests through the portal
- External referral intake with prepopulated referral source
