## UK GDPR

- **Lawful basis**: consent, vital_interests, legitimate_task
- **DSAR**: `POST /v1/admin/dsar` by NHS number
- **Right to erasure**: `POST /v1/admin/erasure` (blocked within 8-year retention)
- **Consent withdrawal**: `PATCH /v1/consents/:id/withdraw`
- **Data minimisation**: push notifications contain only job references

## Data Retention

- Clinical records (PTFs): minimum **8-year** retention
- Soft delete via `deleted_at` — hard delete blocked until retention expires
- `compliance-service` (Docker-only, port 3016) enforces retention policies automatically
- Audit records retained indefinitely (immutable, separate database)

## Compliance Service (Port 3016)

The compliance-service runs in Docker and handles:

| Feature | Description |
|---|---|
| Breach detection | Automated scanning for data breach indicators |
| Retention enforcement | Policy-based lifecycle management for clinical records |
| DSAR processing | Data Subject Access Request fulfilment |
| Right to erasure | Erasure workflow with retention policy checks |
| Data processor registry | Third-party processor tracking and accountability |

<Callout type="info">
  The compliance-service communicates with audit-service (3015) for evidence and
  notification-service (3011) for alerting DPO and administrators.
</Callout>

## Consent Management

Consent records capture:
- `consent_type`: `data_processing`, `data_sharing`
- `consented`: boolean
- `lawful_basis`: required when consent is not given (vital_interests, legitimate_task)
- `representative_name` / `representative_reason`: when patient cannot consent directly
- `withdrawn_at`: timestamp when consent is withdrawn (triggers erasure evaluation)

## Organisation Settings

Dynamic compliance configuration via `OrgSetting` model:
- Settings stored in database with environment variable fallback
- Grouped by category (security, notifications, compliance, billing, etc.)
- Admin-only bulk upsert via `PUT /v1/settings`
- Read by `org-settings` config reader in `@landtts/shared`

## User Preferences

Per-user settings via `UserPreference` model:
- Notification preferences (channels, categories)
- Display preferences (theme, density)
- Shift preferences (default role, availability)
